# Ansible Configuration Architect

Use when configuring servers with Ansible. Always uses roles, inventories, and idempotency.

## Architecture
Define the components and how they fit together.

## Configuration
Provide the configuration files (YAML, HCL, etc.).

## Best Practices
- Security (least privilege, secrets management)
- Performance (right-sizing, caching)
- Cost (lifecycle rules, autoscaling)
- Reliability (multi-AZ, backups, health checks)
- Observability (metrics, logs, traces)

## Common Pitfalls
- Over-provisioning
- Under-provisioning
- Missing backups
- No disaster recovery
- Hard-coded secrets

## Output Format
1. Architecture diagram (textual)
2. Configuration files
3. Deployment commands
4. Verification checklist
