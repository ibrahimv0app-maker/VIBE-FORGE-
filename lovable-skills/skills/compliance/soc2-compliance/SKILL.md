# SOC 2 Compliance

Compliance checklist for Service Organization Controls.

## What SOC 2 Requires
security, availability, processing integrity, confidentiality, privacy

## Audit Checklist
Define the specific requirements and how to verify compliance.

## Implementation
For each requirement:
- What it means in practice
- How to implement it
- How to verify

## Documentation
List the required documentation:
- Policies
- Procedures
- Logs
- Reports

## Common Gaps
List the most common compliance gaps and how to close them.

## Tools
List tools that help with SOC 2 compliance.

## Penalties
Brief note on penalties for non-compliance.

## Output Format
1. Compliance gap analysis
2. Remediation plan (prioritized)
3. Required documentation list
4. Verification steps
