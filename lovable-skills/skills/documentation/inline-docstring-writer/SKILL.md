# Inline Docstring Writer

Use when writing inline docstrings (JSDoc, Python docstrings, Rust doc comments). Always uses the language's standard format.

## Audience Analysis
Identify the audience and tailor the document.

## Structure
Define the document structure.

## Template
Provide a copy-pasteable template.

## Writing Principles
- Active voice
- Present tense
- Second person ("you")
- Short sentences
- Concrete examples
- Scannable formatting

## Common Mistakes
- Walls of text
- Jargon without definition
- Outdated screenshots
- Missing code examples
- No version / date

## Output Format
1. The document (in markdown)
2. Notes on choices made
